# 🐞 Day 13 – Debugging in Python

## 📖 Overview

On **Day 13** of the **100 Days of Code: The Complete Python Pro Bootcamp**, I focused on learning one of the most important programming skills: **Debugging**. Instead of building a new project, this day was dedicated to understanding how to identify, analyze, and fix bugs in Python programs.

I learned that every programmer writes code with mistakes, and debugging is the process of finding and correcting those mistakes so the program behaves as expected.

---

## 🎯 What I Learned

During this lesson, I explored:

* What debugging is and why it is important.
* The different types of programming errors:

  * **Syntax Errors**
  * **Runtime Errors**
  * **Logical Errors**
* Practical debugging techniques used by professional developers.
* How to analyze code instead of guessing where the problem is.
* How to use Python's error messages and debugging tools to locate bugs efficiently.

---

## 🛠 Debugging Techniques Practiced

### 1. Describe the Problem

Clearly identify what the program is supposed to do and compare it with what it is actually doing.

### 2. Reproduce the Bug

Run the program with the same input until the error consistently occurs.

### 3. Play Computer

Trace the program line by line and keep track of variable values to understand how the program executes.

### 4. Fix the Error

Locate the incorrect logic or syntax and apply the appropriate correction.

### 5. Print is Your Friend

Use `print()` statements to display variable values and program flow for easier debugging.

### 6. Use a Debugger

Learned how IDE debuggers can pause execution, inspect variables, and execute code one line at a time.

### 7. Take a Break

Sometimes stepping away from the code for a few minutes helps identify mistakes more easily.

### 8. Ask a Friend

Explaining the problem to someone else (or using the Rubber Duck Debugging technique) often makes the solution clearer.

* Debugging is an essential programming skill.
